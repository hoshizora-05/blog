VScodeでGitHubリポジトリを連携する方法

新規作成の手順と、作成したリポジトリをVScodeと紐づける方法を写真付きで解説します。

なお、GitHubアカウントは作成されていることを前提とします。


まずはGitHubでリポジトリを新規作成

GitHubにログインすると、色々出てくると思います。左ペインのNewから、新規でリポジトリを作成します。
すると、上記写真のような画面が出てくるので、以下の項目に沿って進めてください。


  • Repository name：リポジトリの名前です。なんでもOK
  • Description：リポジトリの説明です。スルーでOK
  • Public or Private：リポジトリの公開範囲を設定します。今回はPublicで進めます。
  • Add a README file：READMEファイルを一緒に作るかどうかの設定です。
  • Add gitignore：使ったことがないので分かりませんが、恐らく言語ファイルのテンプレートを使用するか否かの設定だと思われます。
  • Choose a license：リポジトリの著作物のライセンスを設定します。

見直して良ければ、右下のCreate repositoryをクリックしてください。


VScodeでいろいろする

まず、任意の場所でフォルダを作成してください。その後、作ったフォルダをcodeで開き、左ペインの上から3つめのマークを押すとこのような画面になると思います。
この状態で、左上のリポジトリを初期化するをクリックしてください。

クリックしたら、状態が変化すると思います。この状態で、任意のファイルを新規作成します。

その後、先ほどの場所に戻ると、このように変化していると思います。そうしたら、ファイル名横の＋ボタンをクリックし、ファイルをステージングしてください、

ステージしている状態で、コミットの上にある空欄に任意の文字列を入力し、コミットをクリックしてください。

ここでGitHubに戻り、この画面になっていると思うので、青色の部分の、HTTPSが選択された状態でリンクをコピーしてください。

codeに戻ってきたら、上側のターミナルから、新しいターミナルをクリックすると、写真のような画面になると思います。
その画面で以下のコマンドを叩いてください。

                            
                                git remote add origin "先ほどコピーしたリンク"

これで連携が完了しました。試しにローカルでコミットしたファイルをプッシュしてみます。プッシュに成功していれば、リポジトリのページを更新するとファイルが出現すると思います。

まとめ

分かりづらい解説かもしれませんが、誰かの役に立てれば幸いです。
